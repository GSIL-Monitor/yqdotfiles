Using the counter attribute

When editing a pattern for the "Rename" action, the counter attribute becomes available.

This attribute represents a numerical counter that increases by one each time the action is run. When used in a renaming pattern, it allows you to number files.

In the pattern above, the counter is tacked onto the end of the filename preceded by a dash. If the file name is somefile.doc, then this would create files named like somefile-1.doc, somefile-2.doc, …

Note that the counter is always increasing and will not backtrack or fill in holes in the sequence. Also, counters are specific to the folder the file is in at the time it is renamed. Different folders will have their own counters.
