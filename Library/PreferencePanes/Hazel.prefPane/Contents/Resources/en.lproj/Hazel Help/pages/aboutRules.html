About Rules


The Rules of Rules

Hazel's rule system works similarly to that of Apple Mail. Each folder has a rule list. When Hazel processes a file, it goes through each rule in order and when it finds a match (as specified by the rule's conditions), it runs the actions of that rule. There are some important points to keep in mind:

  • The first rule that matches is the one and only one that is run.
    This means that if it finds a match, subsequent rules in the list are not looked at for that file. Note that this differs from how Apple Mail processes rules. If you want Hazel to continue evaluating rules for that file, you must use the "Continue matching rules" action.
  • If a file is not moved from the folder, Hazel will continue to run the file through the rule list
    This allows you create rules that may not apply to a file now, but will at some future date.
  • Hazel only processes folders and files directly in the folder being monitored.
    If you need Hazel to match against files and folders in subfolders of the main folder, see Processing subfolders
  • If an action fails, subsequent actions are not run for that file.
    If you have five actions on a rule and the third one fails, actions four and five will not get run on that file. Hazel will attempt to re-execute the actions on subsequent runs until it succeeds (as long as the file still matches the rule.
  • If a file matches the same rule it matched last time, it will not re-run its actions
    As an extension of the above. This is done for performance reasons (no use re-running the same actions time and time again if the file keeps matching the same rule). The upshot of this if Hazel colors a file red, for instance, and you change the color, Hazel will not re-color the file red if it matches the same rule. Note that if conditions change and the file starts matching a different rule instead, those actions will get executed.
  • You can exclude files from subsequent rules using the "Ignore file" action
    If you want to make sure certain files are not acted upon by other rules, make the first rule one whose conditions match the files you want to exclude and make the action "Ignore file". Since Hazel only uses the first rule that matches, those files will not be processed by the rules coming after this one.
  • You can have a rule be a catch-all for files that don't match any of the rules.
    If you want to have a default action run on files that don't match any of the other rules, make the last rule match "Any file" and supply whatever actions you like. Since "Any file" matches, as it says, any file, it will match anything that has gotten through all the rules preceding it.
