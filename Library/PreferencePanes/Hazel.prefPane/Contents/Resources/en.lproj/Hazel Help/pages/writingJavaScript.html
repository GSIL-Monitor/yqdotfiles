Writing JavaScript for Hazel

In addition to AppleScript, you can also have Hazel run JavaScript. Just as with AppleScript, you can use them in conditions, via the "Passes JavaScript" condition or in actions, via the "Run JavaScript" action. If you prefer to use AppleScript, see Writing AppleScript.


Writing an JavaScript for use in a rule condition

You can use JavaScripts to have a rule apply your own custom logic to the conditions. Simply have your script return true or false to indicate whether the file passes your script. Keep in mind that since rule conditions are evaluated many times, your script should run fairly quickly otherwise, it may greatly affect rule performance. Also, it is imperative that the script not modify any files or folders underneath the folder Hazel is monitoring.

You can optionally return an object instead of true/false. This allows you to communicate values back to Hazel. Currently the following keys are recognized:

  • hazelPassesScript: Same as returning true or false by itself. Indicates to Hazel whether the file passes the script or not.
  • hazelOutputAttributes: An array of values for the custom attributes that this script exports. The values must be in the order as you specify in the interface. See the section below on Custom Attributes.

For embedded scripts

You can enter JavaScript directly into Hazel as an embedded script. Scripts entered in this fashion are saved with your rules. This means that they are carried with your rules when you transfer your rules around (via export/import).

To show the script editor, click on the "Edit Script" button. Here, you can enter your JavaScript. Note that no function is needed. You can include the body of your script directly in the editor. The variable theFile carries an alias to the file that Hazel will pass to the script.

For external scripts

Hazel requires that the script contain a function called hazelMatchFile. The function takes two arguments which are an alias for the file that matched the rule and an array of input attribute values (see Input Attributes. Below is a template you can use:

function hazelMatchFile(theFile, inputAttributes)
{
	-- Add your logic here.
	-- 'theFile' is an alias to the file that matched.
	-- 'inputAttributes' is an JavaScript array of the values of any attributes you told Hazel to pass in.
	-- Make sure to return true or false (or optionally a record) to indicate whether the file passes this script
}

Writing an JavaScript for use in a rule action

You can use JavaScript to create your own actions to be applied when a rule is executed.

Some notes about writing JavaScripts for the "Run JavaScript" action:

  • If you want the script to terminate unsuccessfully, you must throw an exception. If an error is encountered, Hazel will consider the rule as a whole to have not run successfully and will try to run the rule again later. If no exception is thrown, Hazel will consider the script to have run successfully regardless of what is returned.
  • You can optionally return an object. This allows you to communicate back to Hazel. Currently the following keys are recognized:
    • hazelStop: If set to true, instructs Hazel to stop processing. Subsequent actions for this rule will not get run and the rule as a whole will be considered to have run successfully. You can use this to stop a rule from continuing based on what button a user presses in a dialog, for instance. Hazel will not re-try running this rule.
    • hazelSwitchFile: If set to a valid file, Hazel will apply subsequent actions to this file. You can use this to re-target what file Hazel is operating on, such switching to the parent folder or if the script transforms the file into a new one, you can point this at the new file.
    • hazelOutputAttributes : An array of values for the custom attributes that this script exports. The values must be in the order as you specify in the interface. See the section below on Custom Attributes.
  • If anything besides an object is returned, that value is ignored.

For embedded scripts

You can enter JavaScript directly into Hazel as an embedded script. Scripts entered in this fashion are saved with your rules. This means that they are carried with your rules when you transfer your rules around (via export/import).

To show the script editor, click on the "Edit Script" button. Here, you can enter your JavaScript. Note that no function is needed. You can include the body of your script directly in the editor. The variable theFile carries an alias to the file that Hazel will pass to the script.


For external scripts

For external scripts, Hazel requires that the script contain a function called hazelProcessFile. The function takes two arguments which are an alias for the file that matched the rule and an array of input attribute values (see Input Attributes. Below is a template you can use:

function hazelProcessFile(theFile, inputAttributes)
{
	-- Add your logic here.
	-- 'theFile' is an alias to the file that matched.
	-- 'inputAttributes' is an JavaScript array of the values of any attributes you told Hazel to pass in.
	-- Make sure to throw errors if you do not want Hazel
	--   to consider this action to have run successfully.
}

Input Attributes

Scripts take in an array of input attribute values as its second argument.

To specify these attributes, click on the Info button: in the JavaScript condition or action.

There, on the right side, you can specify which attributes to pass into your script.

Most of the built-in attributes, as well as Spotlight attributes, are available here. Any custom attributes that are defined before the condition or action are also available.

When your JavaScript is run, Hazel will pass in the values for those attributes into your script as an array. They will appear in the order specified.


Custom Attributes

When writing a script, you can export custom attributes. These attributes can then be made available for any patterns for actions that come afterwards. Also, if the script is used in a condition, the resulting custom attribute can be used in subsequent conditions just like built-in attributes.

To export your attributes, just return an array in an object as described above.

To make these attributes accessible to your rule, click on the Info button: .

There, on the right side, you can add the names of any attributes your script exports. The order of the values you return in the script should match the order of the attributes you define here. These values will then be substituted wherever the attribute is used. Note that you cannot use a custom attribute before it is defined so make sure the JavaScript condition/action appears before any condition/action that will use those tokens.
