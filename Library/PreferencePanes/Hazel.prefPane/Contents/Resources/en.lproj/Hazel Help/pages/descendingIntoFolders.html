Processing subfolders

Normally, Hazel treats files and folders the same. They are considered one unit when it comes to filtering and applying actions. For example, the "Move" action will move the whole folder.

Sometimes, though, you want Hazel to apply its rules to the contents of a subfolder. There are two ways to accomplish this. You can (a) add the subfolder to Hazel just like any other folder or (b) use the "Run rules on folder contents" action. Method (a) is fairly straightforward as you just treat a subfolder as you would any other folder that you have Hazel monitoring. But what about method (b)?

The "Run rules on folder contents" action does just that. When a folder matches the conditions of the rule, the action will take the rule list for the current folder and apply them to the files and folders in the folder that just matched. Think of it as a hand-off from the parent folder to the child folder. Depending on how you set up the conditions, this hand-off can be performed over and over therefore allowing Hazel to descend further down into subfolders.


Limitations with using "Run rules on folder contents"


Note that this can get tricky so here are a few things you should keep in mind:

  • The action will fail if is applied to anything besides a folder. Make sure you use a "Kind is Folder" condition to ensure that non-folders do not match. If a regular file matches the rule, it will not be able to match any other rules which is probably not what you intended.
  • Descending into a folder goes from top down. The rule matches at the parent folder and gets handed off from parent to child. You need to be careful how you structure your conditions/rules such that it will go into the folders you want but not into the ones you do not.
  • Descending into large tree of folders is very resource intensive. Try to structure the rules such that they do not descend too deeply.

When to add the folder to Hazel explicitly


  • When the folder is somewhat permanent and known ahead of time.
  • When you require Hazel to monitor changes to it directly.
  • When there are a few of them.

When to use the "Run rules on folder contents" action.


  • When the folders are created dynamically or the exact name are not known ahead of time.
  • When there are many of them.
