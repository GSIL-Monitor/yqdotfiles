Rule Reference Manual

This section describes all the different file attributes and actions that are available in the Hazel rule interface.


File Attributes


Name
This is the file's name, without the extension. So, if you have a file called "example.jpg", this attribute would match against "example".
Extension
This is the file's extension, which is the part of the file name after the last dot ("."). The file "example.jpg" would have an extension "jpg". Note that if you have Finder hide extensions, then the extension may not be visible to you.
Full Name
This represents the full name of the file, which is the name plus the extension.
Date Added
This is the date when the file was added to the current folder. This attribute is useful for determining how long a file has been sitting in a particular folder.
Date Created
This is the date the file was created.
Date Last Modified
This is the date the file was last changed.
Date Last Opened
This is the date the file was last opened.
Date Last Matched
The date when the file last matched a rule and had its actions successfully run against it. This attribute is useful for cases where you want to specify "since last time>
Kind
This indicates the type of file, such as a movie or a folder. Selecting "Other" as a kind will bring up a list of all the available file types on your system.
Tags
This is a list of tags that can be assigned to a file.
Color Label
This is the color label, if any, set on the file. This will match any of the tags which are currently associated with the color.
Comment
This is where you can add your own notes to a file. You can edit these comments in Finder via the "Get Info" option.
Size
This is the size of the file. This is the actual size of the file which may be less than the amount of space it takes up on disk.
Contents
The text content of the file. If specifying "contain/do not contain", you can match any words that Spotlight has indexed for this file. In a nutshell, if you can find the file via Spotlight using a certain word, that word will be contained in the file's contents.

If using "contain match/do not contain match", you can match against a pattern, and optionally grab text so that it can be used in your actions. For more details on creating match patterns, see Editing match patterns.
Source URL/Address
This is the URL or email address the file came from. Note that only certain programs store this information, such as Safari and Apple Mail.
Subfolder Depth
How many folders deep, starting from the folder being monitored. This is only useful if there is a rule set up to descend into subfolders using the "Run rules on folder contents" action. A subfolder depth of 0 indicates the top level folder (the one Hazel is set to monitor).
Sub-file/folder Count
How many files/folders are contained by the folder being processed. This only counts the number of items immediately underneath the folder, not those contained in subfolders of this one. If the current item being processed is a file, then this will always be zero. This is similar to Spotlight's "Number of Items" attribute except that this does not include hidden files in the count.
Passes AppleScript
Allows you to provide an AppleScript to evaluate the file. For details on writing AppleScript for Hazel, see Writing AppleScripts
Passes shell script
Allows you to provide a shell script to evaluate the file. See Writing Shell Scripts for more information.
Any File
This attribute matches all files. This is useful for use in the last rule in a list to apply to any files that do not match any of the previous rules.
Other
This attribute will allow you to select any Spotlight attribute in the system. Once selected, you can filter on the attribute as if it were one of Hazel's built-in attributes.

Actions


Move
This moves the file/folder to the specified destination folder. The "Throw away if duplicate" option specifies whether you want the item to be thrown away if it already exists at the destination. If an item with the same name exists at the destination folder and it is not a duplicate, you can specify whether a number should be added to the end of the name, that the new item should replace the old one, or the new one thrown away in favor of the old one. You can also specify that some of the folder structure be copied over as described in Copying folder structure. This action will fail if the source and destination folders are the same.
Copy
This copies the file/folder to the specified destination folder. If copying a folder, the folder and all its contents (and all its contents' contents and so on) are copied. The "Do not copy if duplicate" option specifies whether you want the file to not be copied if it already exists at the destination. If an item with the same name exists at the destination folder and it is not a duplicate, you can specify whether a number should be added to the end of the name, that the new item should replace the old one, or to keep the old one. You can also specify that some of the folder structure be copied over as described in Copying folder structure. This action will fail if the source and destination folders are the same. Note that subsequent actions in the same rule apply to the copy and not the original.
Rename
Renames the file/folder. You can specify a pattern to use for the renaming. See Editing patterns on how to edit the pattern.
Sort into subfolder
Sorts an file/folder into a subfolder based on a specified pattern. If the folder does not exist, it will be created as needed. See Editing patterns on how to create the pattern. Note that you can indicate additional levels of folders by using the ▸ token. Also, the "anything" attribute (…) can be used to match against partial names of existing folders. When using this, keep in mind that the folder cannot be created for you.
Sync
Syncs the file/folder to the destination. This is similar to copy but it will only copy those files that have changed. Note that the sync is one-way. Files changed at the destination will not get copied back.
Upload
Uploads the file/folder to a server. Uploads can be done via FTP, SFTP, and WebDAV. Hazel will ask for your login credentials (if you do not have these, ask the server's administrator) and then connect, allowing you to choose a destination folder. Your credentials will be securely stored in your keychain so that Hazel can log in without prompting when the rule runs. You can also specify that some of the folder structure be copied over as described in Copying folder structure.
Add tags
Adds tags to the file/folder. You can also add Dynamic Tags where the tags are created on the fly based on file metadata and custom attributes you create. See Editing patterns for more details on how to use patterns to create Dynamic Tags.
Remove tags
Removes the specified tags from the file/folder. You can have all the tags on a file removed by using the special “any existing tags” item. You can also specify Dynamic Tags where the tags are created on the fly based on file metadata and custom attributes you create. See Editing patterns for more details on how to use patterns to create Dynamic Tags.
Set color label
This sets the color label on the file/folder. Select the "X" if you want the color label to be removed. This sets the all tags associated with the given color and will remove any other colored tags from the file.
Add comment
This adds a comment to the file/folder which is searchable via Spotlight. If you include the "Comment" attribute in your pattern, it will include the pre-existing comments in your new comment, otherwise the pre-existing comment will be replaced.
Toggle extension
This sets whether the file's extension is hidden or visible.
Archive file
Converts the file/folder to a zip file. This is the same type of archive you would get using the "Compress" feature in the Finder. Note that subsequent actions in the same rule will operate on this new archive; the original item is put in the Trash.
Unarchive file
If the file is an archive, the file is unpacked. Note that subsequent actions in the same rule will operate on the unpacked file/folder; the original archive file is put in the Trash. This action will fail if the file is not an archive.
Open file
Opens a file. You can specify a specific application or use "Default Application" to open the file with the application that would normally open it when you double-click on it. Some caution should be exercised when opening files from untrusted sources; structure your rule conditions appropriately. Also, if you use this action, you should disable any browser options that will cause it to open files otherwise you may end up with files opening twice.
Show in Finder
Opens a new Finder window with the file/folder selected.
Make alias
Creates an alias to the file/folder in the specified folder. Aliases are useful as shortcuts to other files.
Import into iTunes
Imports the file into iTunes. The file itself is not moved anywhere so depending on your iTunes settings, you may want to move it to its proper place beforehand (if you have iTunes set to not copy files) or throw it away afterwards (if you have iTunes set to copy files). Note that iTunes will be launched if it is not already running.
Import into Photos
Imports the file into Photos. The file itself is not moved anywhere so depending on your Photos settings, you may want to move it to its proper place beforehand (if you have Photos set to not copy files) or throw it away afterwards (if you have Photos set to copy files). Depending on your Photos preferences, Photos may or may not copy files it imports. Note that Photos will be launched if it is not already running.
Import into iPhoto
Imports the file into iPhoto. The file itself is not moved anywhere; iPhoto copies files it imports. Therefore, it is up to you to set up a subsequent action that moves the file where you want it. Note that iPhoto will be launched if it is not already running.
Import into Aperture
Imports the file into Aperture. You can specify the folder or project to import the file into. You can also tell Aperture to copy/move/leave the file when importing. Note that Aperture will be launched if it is not already running.
Run AppleScript
Allows you to trigger an AppleScript. For details on writing AppleScript for Hazel, see Writing AppleScripts.
Run Automator workflow
Automator actions can also be triggered by rules. Make sure your Automator workflow takes a file as input.
Run Shell Script
Runs a shell script or any command-line program. See Writing Shell Scripts for more information.
Run rules on folder contents
This causes Hazel to run the full rule list on the files and folders contained in this one. You should make sure you have a condition specifying "Type is Folder" as this action will fail if run on a file. If sub-folders match a rule with this action, Hazel will continue to descend as needed. Be careful about allowing Hazel to descend too far as this will delay processing of subsequent files. For more information, see Processing subfolders
Continue matching rules
This tells Hazel to continue matching against subsequent rules instead of stopping. Normally Hazel will stop once it finds a match but this will indicate that rule evaluation should continue. Note that you cannot continue evaluation if the file is moved out of its folder. Therefore, you cannot use this action in conjunction with the Move and Sort into Subfolder actions.
Display notification/Display Growl notification
This action allows you to display a notification on screen. You can even set a sound to be played, accessible when clicking the options button.

See Editing patterns on how to edit the pattern.
Ignore File
This action does nothing. It is useful for ensuring that certain files do not get processed. For instance, you can set a rule at the beginning of a list to exclude certain files from any subsequent rules. This action only makes sense when it is the only action in a rule.
