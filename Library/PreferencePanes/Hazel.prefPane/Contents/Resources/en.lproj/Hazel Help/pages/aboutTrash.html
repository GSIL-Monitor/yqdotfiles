Time to take out the Trash…


In addition to file organization, Hazel also can manage your Trash by deleting files from the Trash based on different settings. It is important to note that deletion here means permanent deletion so it is wise to set the limits higher/longer than you think when you first try it out.

  • Delete files sitting in the Trash too long
    Hazel can record how long files have been sitting in the Trash and when they have been there longer than a time you set, they are removed.
  • Keep the total Trash size under a set limit
    This tells Hazel to keep the total size of all files in the Trash under a limit you set. If a file is added and the total size goes over the limit, Hazel will remove files in the order in which they entered (so first in, first out) until the total size is under the limit.

Oversize files

An oversized file is a file that, by itself, exceeds the Trash size limit. If you have Hazel enforcing a size limit, you have two options for deailng with these types of files:

  • left as is
    The file will be left in the Trash but Hazel will ignore it when calculating the total Trash size (otherwise, the Trash will always be over the limit). If you have the "Delete files sitting in Trash for more than…" option checked above, the file will eventually be erased when its time limit is reached. Otherwise, it will only be removed if you manually empty the Trash. The text below the option will indicate what behavior to expect based on your settings.
  • deleted immediately
    As it says, the file will be deleted immediately. There will be no chance for recovery.

Normal vs secure deletion

Normally, when files are deleted, their data is left intact. Deletion in this case merely makes the space the file is occupying available to be re-used. This is why file recovery and undelete tools exist as they can search the disk for these stranded files.

The upshot of this is that if you have sensitive data, this data is still on your drive for some period of time. If security is a concern, you can tell Hazel to securely delete files. This is equivalent to shredding a document in the physical world. Hazel uses a 7-pass US Department of Defense algorithm to overwrite the file to ensure that the data is not recoverable. Note that this is the same mechanism that is used if you use Finder's "Secure Empty Trash" option.

Which should you choose? Here's a breakdown of the pros and cons.

Normal deletion:
+ Quick
+ Files may be able to be recovered through special software
- Data may be available on the drive long after you have deleted the file

Secure deletion:
+ Data is overwritten and unavailable to any snoopers.
- Slow and resource heavy
- Files cannot be recovered, even with special software.

If in doubt, use the setting that corresponds to how you emptied the Trash before Hazel. If you just used the "Empty Trash" option in Finder then use normal deletion and if you used "Secure Empty Trash" then use secure deletion.

App Sweep

App Sweep is a feature where Hazel watches the Trash for thrown away applications and other software. When you throw away a piece of software, App Sweep can check for other support files and offer to throw them away as well. See the section on App Sweep for more details.
